# Documentation Hub

`docs/` ディレクトリは、コードから独立した一次情報（要求、方針、手順、テンプレートなど）を体系化するためのハブです。`README.md` や `AGENTS.md` に収まらない詳細はここに配置します。

## 構成ガイド

- `glossary.md`: 頻出語彙の定義。
- `dev-practices/`: 開発時に参照する詳細手順（フロー、コミット、lint、運用など）。
- `policies/`: ドキュメント配置、命名、承認モデルなどのポリシー集。
- `templates/`: テンプレートやフォーマット（例: BDD シナリオ雛形）。
  - 進行・承認ヘッダ（通常応答向け）の仕様は `AGENTS.md` の該当節を参照してください（テンプレートは提供しません）。
- `product/`: プロダクト全体のコンセプトと共通前提。
- `mcp/`: MCP サーバーやツールの手順。

- ポリシー個別文書（project-basics, approval など）は `policies/README.md` からトップダウンで辿ってください（厳格トップダウン運用）。

必要に応じてフォルダを増やし、曖昧な名前は避けてください。詳細は `policies/README.md` → `policies/documentation.md` を参照します（上位 README から下位へ）。

## 利用ルール

- 新しい一次情報は必ず `docs/` に追加し、`README.md` や `AGENTS.md` には要点と導線だけを残します。
- 作業前に関連ドキュメント（特に `dev-practices/` と `policies/` 配下）を確認し、最新の運用と前提を把握してください。
- ドキュメントを更新したら、関連するテストや実装も整合させます。

## 運用ガイド

- 変更はタスク提案に含め、合意後に更新します。Boy Scout Rule に従い、小さな改善を継続してください。
- 外部調査や議事録は要点のみを一次情報として記録します。
- 機密情報の記載は禁止です。認証情報は安全な保管場所を利用してください。
