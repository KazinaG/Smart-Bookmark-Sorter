# 開発プラクティス集

開発時に参照する詳細な運用ドキュメントをまとめています。作業前に該当ドキュメントを必ず確認し、提案内容に反映してください。

- `process.md`: トップダウン開発フローとワークツリー運用
- `commit.md`: コミット前後の手順とメッセージ規約
- `linting.md`: lint と検証の既定ループ
- `operations.md`: コマンド実行・タイムアウト・WT 基本
- `testing.md`: Regression-first × BDD を前提としたテスト運用ガイド

承認モデルや開発方針は `docs/policies/approval.md` および `docs/policies/development-principles.md` と合わせて参照してください。

必要に応じて新しいプラクティスを追加し、`docs/README.md` からリンクしてください。
