# プロジェクト前提と運用前提

## プロジェクト前提（個人開発）

- 本プロジェクトは個人開発であり、Codex CLI（エージェント）との協働を前提とします。
- 「チーム／社内」を前提とした表現やフローは原則用いず、必要最小の運用とします。
- 課題管理は GitHub Issue を基本とし、現時点では PR を作成しません。必要になった場合はガイドラインを更新し、以降の運用を明示します。
- GitHub 操作の手順や `gh` コマンドの使い方は `docs/dev-practices/operations.md#github-オペレーション` を参照してください。
- リポジトリ編集作業は必ず git worktree の専用ブランチで行い、`main` への直接コミットは原則禁止です。
- 例外（要明示承認）: ユーザーの明示要請がある場合のみ個別に許可します。
- ブランチ命名: `feat/<slug>` / `fix/<slug>` / `docs/<slug>` / `chore/<slug>`。

## 運用前提（Codex CLI）

- 既定の実行環境（DevContainer 前提）
  - `remoteUser = "node"` を使用する
  - `workspaceFolder = "/workspace"` を使用する
  - `.codex` を `/home/node/.codex` にマウントし、設定と履歴を永続化する
  - 承認モデル・サンドボックス・ネットワーク可否はハーネス設定に従う
- 上記は運用上の前提であり、最終的には本ガイドラインの規約が優先されます。
- 反映操作の承認境界は `docs/policies/approval.md` を正本とします。
  - 開始前はステージング・コミット・Push を行いません。
  - `実装OK: <scope>` 後は、`worktrees/<branch>` 内の add/commit は Fast-Path で実施可能です（Push / `main` 取り込みは別承認）。

## 秘密情報の取り扱い

- リポジトリ・ログ・チャットには原則として秘密情報を記載しません。
- サービスアカウント鍵などの認証情報は、バージョン管理から除外した安全な場所で保管します。
